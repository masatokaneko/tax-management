# freee法人税申告システム ブレインストーム

**日付**: 2026-02-26
**ステータス**: ブレインストーム完了 → 計画策定待ち

---

## What We're Building

freeeの会計データを用いて、法人税務申告をエンドツーエンドで自動化するシステム。
Claude Code + MCP をインターフェースとし、税務申告専用MCPサーバーを構築する。

### スコープ

| 項目 | 内容 |
|------|------|
| **対象税目** | 法人税・消費税・地方税・法人住民税・事業税・事業所税・償却資産税 |
| **対象別表** | 別表一〜十六（全別表） |
| **初期ターゲット** | 株式会社Scalar（freee会社ID: 1356167） |
| **将来展開** | 税理士・中小企業向けSaaS化も視野 |
| **出力形式** | eLTAX電子申告（XBRL形式） |
| **AI役割** | データ取得→税務調整推定→申告書作成→整合性チェック→提出（AI推定+ユーザー確認のハイブリッド） |

---

## Why This Approach（MCPネイティブ）

### 選定理由

1. **最速プロトタイプ**: Web UI開発不要。Claude Codeがそのままインターフェース
2. **既存インフラ活用**: freee MCP（@him0/freee-mcp）やfreee-analysis-systemの認証・APIクライアントをそのまま利用
3. **AIの柔軟性**: Claudeの税務知識を直接活用。ルールベースでは対応困難な税務判断をAIが補完
4. **自社利用に最適**: まず自社（Scalar）の申告に特化するため、SaaS化の制約は当面問題なし

### アーキテクチャ概要

```
┌─────────────────────────────────────────────────┐
│ Claude Code (ユーザーインターフェース)            │
│   - 対話的に税務申告を進行                      │
│   - 税務調整のAI推定・ユーザー確認              │
│   - 申告書の最終レビュー・承認                  │
├─────────────────────────────────────────────────┤
│ 税務申告 MCP Server (Node.js/TypeScript)        │
│   ├─ 別表計算ツール群 (別表1〜16)               │
│   ├─ 消費税計算ツール (一般・簡易)              │
│   ├─ 地方税計算ツール (住民税・事業税)          │
│   ├─ 税務調整ツール (加算・減算項目管理)        │
│   ├─ 整合性チェックツール                       │
│   ├─ eLTAX/XBRL出力ツール                      │
│   └─ 前期データ管理ツール                       │
├─────────────────────────────────────────────────┤
│ 既存 freee MCP Server (@him0/freee-mcp)         │
│   ├─ 仕訳データ取得                             │
│   ├─ 試算表 (BS/PL) 取得                        │
│   ├─ 勘定科目・取引先マスタ取得                 │
│   └─ 月次推移表取得                             │
├─────────────────────────────────────────────────┤
│ ローカルデータストア                             │
│   ├─ 税務調整データ (JSON/SQLite)               │
│   ├─ 前期申告データ                             │
│   ├─ 税率テーブル・税法マスタ                   │
│   └─ 申告書生成履歴                             │
└─────────────────────────────────────────────────┘
```

### 不採用アプローチ

- **A: AIエージェント + Web UI**: 強力だが、Web UI開発のオーバーヘッドが大きい。自社利用段階では過剰
- **B: Pythonモノリス**: 既存資産を直接拡張できるが、AIエージェントの柔軟性が低い。税務判断の曖昧さに対応しにくい

## Key Decisions

### 1. MCPサーバーの言語: TypeScript
- 既存のquickbooks-MCP-Scalarと統一
- MCP SDKのTypeScriptサポートが最も充実

### 2. データストア: SQLite + JSON
- 税務調整データ・前期申告データの永続化にSQLite
- 税率テーブル・税法ルールはJSONファイルで管理（バージョン管理しやすい）

### 3. eLTAX連携: XBRL形式出力
- eLTAXの電子申告はXBRL形式
- まずXBRLファイル生成→手動アップロードから開始
- 将来的にeLTAX Web APIによる自動提出を検討

### 4. 税務調整のアプローチ: ハイブリッド
- AIがfreeeデータから税務調整項目を推定（交際費、寄付金、減価償却超過額等）
- ユーザーが確認・修正してから申告書に反映
- 推定ルールは段階的に精度向上

### 5. 別表の実装順序
- Phase 1: 別表一・四・五（コア）→ 所得金額・税額計算の骨格
- Phase 2: 別表二・十五・十六 → 損益・交際費・寄付金
- Phase 3: 残りの別表 + 消費税 + 地方税
- Phase 4: eLTAX電子申告連携

## Open Questions

1. **eLTAXのXBRL仕様**: 法人税申告書のXBRLタクソノミーの詳細仕様の調査が必要
2. **税法改正への追従**: 毎年の税制改正にどう対応するか（税率テーブルのバージョニング等）
3. **前期データの初期投入**: 初回利用時、前期の申告書データをどう取り込むか
4. **消費税のインボイス対応**: 適格請求書等保存方式への対応範囲
5. **地方税の自治体別差異**: 法人住民税・事業税の税率は自治体ごとに異なる。マスタ管理方法
6. **freee APIの制限**: APIレートリミット、取得可能データの粒度の確認
